代码规范

发表于 2018-07-31 | 分类于 coding

项目组内有一套统一的代码规范非常有利与项目维护

个人vscode配置

  • 在vscode中安装 eslint 插件

  • code - 首选项-设置置

这样vscode可以在保存 （command+s）的时候自动修正代码规范

这里列出几个常用的配置

rules

  • quotes

引号类型： 单引号

  • no-unused-lets

不能有声明后未被使用的变量或参数

  • 同一模块有多个导入时一次性写完

no-duplicate-imports

默认配置项

项目中eslintrc文件中有配置项

表示已经使用了 eslint-config-standard 这个中配置的规则

  • 使用两个空格进行缩进
  • 关键字后面加空格
  • 函数声明时括号与函数名间加空格
  • 始终使用 === 替代 ==
  • 字符串拼接操作符 (Infix operators) 之间要留空格
  • 逗号后面加空格

  • 不允许有连续多行空行

  • 每个 let 关键字单独声明一个变量

  • 始终将逗号置于行末

  • 除非key包含 - 之类的特殊符号，否则一律不添加单引号
  • 键值对当中冒号与值之间要留空白
  • 不要定义冗余的函数参数

  • 避免不必要的布尔转换

  • 对象属性换行时注意统一代码风格

  • 代码块中避免多余留白
  • 注释首尾留空格
  • 检查 NaN 的正确姿势是使用 isNaN()

  • 不要使用分号

  • 不建议使用var
